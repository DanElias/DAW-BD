PREGUNTAS E INVESTIGACIÓN
¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
Por obvias razones es mejor validar los datos usando JavaScript porque el tiempo de respuestas es mucho menor. Al validar con javascript, en el momento se llama a las funciones que validan los datos introducidos sin tener que acceder a otra página o esperar la respuesta de algún elemento externo. Si el servidor tuviera que validar la respuesta, el usuario tendría que esperar más tiempo, además de que una doble validación dada por el JavaScript y por el servidor ayudará a tener una mejor integridad de los datos.[1]
¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
Al parecer hay una manera en la que se puede deshabilitar el javascript del navegador y ya con eso no se checan las validaciones que estén en un js. También se puede colocar código JavaScript en la URL para modificar lo que la página realiza.[2]
Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
La validación no sólo se debe hacer en JavaScript, sino que también en el lado del servidor. De hecho basta con la validación del servidor para tener una página web segura, pero el agregar validaciones en JS permite que la experiencia de usuario sea mucho mejor y la página más responsiva, además de que sirve como un filtro de seguridad extra.[3]
