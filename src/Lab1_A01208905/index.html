Preguntas e investigación
URL
Uniform Resource Locator. Secuencia de caracteres que sigue un estándar y que permite denominar recursos dentro de Internet para que puedan ser localizados. Posibilita la asignación de una dirección exclusiva a un recurso que se encuentra disponible en el espacio virtual.[1]
MIME
Multi-Purpose Internet Mail Extensions. Son extensiones del protocolo de correo electrónico (SMTP) que permiten a las personas intercambiar diferentes tipos de archivos como audio, video, imágenes, el texto ASCII manejado por el protocolo SMTP y otros tipos. Las etiquetas MIME se utilizan para identificar un tipo de datos. Se utiliza para que el software pueda saber cómo manejar los datos.[2]
Métodos HTTP
Conjunto de métodos de petición que indican que se va a realizar con un recurso determinado.[3]
Códigos de errores HTTP
Indican que ha ocurrido al tratar de cargar una página.

-2xx: no existen errores.
-200: la página ha cargado correctamente

-3xx: redirección.
-301/302: se realiza un redirección de una página a otra.

-4xx: errores del cliente.
-400: solicitud tiene sintaxis incorrecta.
-403: está prohibido el acceso a esa página.
-401: se prohíbe el acceso pero se muestra un formulario para tener acceso.
-404: la página que se quiere cargar no se ha encontrado.
-406: la página regresó un código que no se puede interpretar por el navegador.

-5xx: error del servidor.
-500: error interno, la página web tiene errores en su código.
-504: el tiempo de espera se ha agotado.
-509: se ha superado el límite de ancho de banda disponible.[4]
XML
es una herramienta independiente de software y hardware para almacenar y transportar datos. Sólo es información con etiquetas, no realiza nada.[5]
XHTML
EXtensible HyperText Markup Language. Es igual a HTML pero con ciertas diferencias. Es más estricto ya que los elementos XHTML deben de estar debidamente acomodados, siempre se deben de cerrar y estar escritos en minúscula. Los atributos deben de estar escritos en minúsculos, los valores deben de estar entre “ “ y la minimización está prohibida. Finalmente el XHTML Doctype es obligatory al igual que las etiquetas html, head, title, body.[6]
HTML5
Es la última versión de Hypertext Markup Language, el código que describe las páginas web. Fue diseñado para ofrecer casi todo lo que se requiera hacer en línea sin requerir software adicional en el navegador. Puede animar aplicaciones, mostrar música, películas y crear aplicaciones web más complejas.[7]
JS
JavaScript es un lenguaje de programación para web, el cual puede actualizar y cambiar tanto HTML como CSS, además de calcular, manipular y validar datos.[8]
PHP
Hypertext Pre-processor es un lenguaje de scripting de código abierto ampliamente utilizado. Los scripts son ejecutados en el servidor. Además es gratis y es utilizado para crear aplicaciones web.[9]
JSP
Java Server Pages es una tecnología de programación que se ejecuta en el servidor y que permite la creación dinámica e independiente de aplicaciones web. JSP tiene acceso a toda la familia de APIs de Java.[10]
AJAX
Asynchronous JavaScript and XML es una técnica para crear aplicaciones web mejores, más rápidas e interactivas con la ayuda de XML, HTML,XHTML, CSS y Java Script. Con AJAX, JavaScript hará una petición al servidor, interpretará los resultados y actualizará la pantalla actual. El usuario no se da cuenta de que algo fue transmitido al servidor.[11]
¿Cuál es la diferencia entre Internet y la World Wide Web?
El Internet es una red amplia y compleja de computadoras conectadas entre sí. Este término comenzó a utilizarse en 1969 por ARPA (Advanced Research Projects Agency) World Wide Web es una colección de páginas web que siguen el protocolo HTTP a las que se puede acceder a través de Internet desde cualquier parte del mundo. [12]
¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
-GET: para recuperar datos o solicitar una representación de un recurso.
-HEAD: es igual al GET pero sólo transfiere la línea de estatus y la sección del header.
-POST: envía información o datos a un servidor.
-PUT: reemplaza todas las representaciones actuales del recurso con el contenido que se acaba de subir.
-PATCH: hace cambios parciales al recurso.
-DELETE: remueve todas las representaciones actuales del recurso.
-CONNECT: establece un tunel de conexión con un servidor.
-OPTIONS: describe las opciones de comunicación con el recurso.
-TRACE: hace una prueba loop-back mandando un mensaje al recurso.[3]
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
POST, ya que este funciona para enviar información o datos a un servidor. [3]
¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
GET. [3]
Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
Los códigos 2xx significan que no ocurrió ningún error. [3]
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
Los códigos 4xx son errores del cliente, por lo tanto el usuario fue quien intentó acceder a una página que no existe. No es responsabilidad del desarrollador corregir el sitio web. [3]
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
Los errores 5xx son errores en el servidor, el 500 significa error en el código de la página web, por lo tanto el desarrollador es el responsable de corregir el sitio web. [3]
¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
Eso significa que van siendo eliminados. Algunas etiquetas y atributos están en desuso porque en realidad ya no sirven para un propósito, la mayoría está en desuso en favor de las hojas de estilo. Algunos ejemplos son align, alink, link, vlink, name, archive, codebase, entre otros. [13]
¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
El Doctype para HTML4 era más largo y en HTML5 solo basta con poner
En HTML4 solo se usaban div con un id, pero en HTML5 existen tags como header, nav, section, article, aside y footer que mejoran la experiencia de desarrollo web y permiten a los navegadores identificar los elementos del HTML de manera más sencilla.En HTML5 en los forms ahora si existe el atribute Required para que el usuario llene los campos requeridos.
Otra diferencia es que HTML5 tiene manejo de errores consistente y definido. También HTML5 tiene elementos como canvas, video y audio que permiten crear aplicaciones web de manera más sencilla.[14]
¿Qué componentes de estructura y estilo tiene una tabla?

Etiquetas:

-thead : header de la tabla.
-tr : renglones.
-th : se usa para las celdas que son encabezados.
-td : se usa para la celda normales con información.
-tfoot : footer de la tabla.
-tbody : body de la tabla.

Estilos:

-border-spacing : espaciado entre celdas y renglones.
-border-collapse: collapse - quitar el espaciado.
-colspan : el número de columnas que se extiende una sola celda en un renglón.
-rowspan: el número de filas que se extiende una sola zelda en una columna.
-white-space: no wrap : la tabla se hará más ancha conforme se ocupe el espacio en una celda, se podrá salir del container.[15]
¿Cuáles son los principales controles de una forma HTML5?
-Text Fields -input-: obtener texto del usuario. Hay de diferentes tipos: color, email, range, time, date, month, search, url, datetime, number, tel, week.
-Text Areas -textarea-: acepta texto más largo y párrafos.
-Radio buttons: sólo permiten al usuario seleccionar una opción.
-Check Boxes: los usuarios pueden seleccionar varias opciones.
-Drop-Down Lists: Dar una lista de opciones al usuario: -select- -option- -/option- -/select-. Puede agregarse el atributo multiple para seleccionar varias opciones.
-Submit: procesa la información dada después de haber llenado el form.
-Puede ser -input- o -button-
-Hidden input: manda información necesaria para procesar el form pero sin importancia para el usuario al servidor sin que el usuario se de cuenta.
-File Input: Para adjuntar un file al form.
-Label: Texto que describe los elementos a llenar del form.
-Fieldset: organiza controles y labels del form en secciones.
-Legend: es el label pero del Fieldset.
-Disabled: es un atributo booleano que hace que usuario no pueda tener interacción con un elemento o control del form.
-Placeholder: es un elemento que da al usuario una pista o ejemplo de input debe de proporcionar.
-Required: obliga al usuario a llenar este elemento del form antes de dar click en submit.[16]

¿Cuál es el ciclo de vida y desarrollo de los sistemas de información?

-Planificación: delimitar el ámbito del proyecto, realizar estudios de viabilidad, analizar riesgos, estimación de costos y tiempos, planificación temporal, asignación de recursos.

-Análisis: elicitación de requerimientos, modelación del sistema, análisis de requerimientos.

-Diseño: arquitecturas multicapa, diseño de la base de datos y aplicaciones, diseño lógico de la pliación, diseño de la interfaz de usuario, el acceso a los datos.

-Implementación: control de versiones, programación, selección del entorno de desarrollo, selección de algoritmos y estructuras de datos adecuadas,dar formato correcto al código.

-Pruebas: pruebas de unidad, de integración, alfa, beta, test de aceptación, búsqueda de errores.

-Instalación: planificar el entorno en donde el sistema va a funcionar tanto hardware como software, equipos necesarios y su configuración física, redes de interconexión, acceso a sistemas externos, sistemas operativos, versiones.

-Uso y mantenimiento: eliminar defectos detectados durante el uso del sistema, adaptarlo a nuevas necesidades y añadir funcionalidades.[17]
